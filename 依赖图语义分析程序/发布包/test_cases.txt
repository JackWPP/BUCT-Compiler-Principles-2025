# 依赖图语义分析程序测试用例

## 测试用例1：简单表达式
输入串: id + id
语法分析结果: 5 3 1 5 3 0
期望结果: 成功构建依赖图，无循环依赖

## 测试用例2：复杂表达式
输入串: id * id + id
语法分析结果: 5 5 3 2 5 3 0
期望结果: 成功构建依赖图，正确计算属性值

## 测试用例3：带括号的表达式
输入串: ( id + id ) * id
语法分析结果: 5 5 3 0 4 5 3 2
期望结果: 成功构建依赖图，处理嵌套结构

## 测试用例4：单个标识符
输入串: id
语法分析结果: 5 3 1
期望结果: 简单依赖图，直接传递属性值

## 测试用例5：错误的语法分析结果
输入串: id + id
语法分析结果: 999 888 777
期望结果: 报告错误，产生式编号不存在

## 说明
- 产生式编号对应关系：
  0: E -> E + T
  1: E -> T
  2: T -> T * F
  3: T -> F
  4: F -> ( E )
  5: F -> id

- 语法分析结果为自底向上归约的产生式序列
- 依赖图应正确反映属性之间的依赖关系
- 拓扑排序应给出正确的计算顺序
